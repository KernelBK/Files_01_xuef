# 关于设计模式的讨论

## 动态语言与设计模式

动态语言不需要这 23 种设计模式，它们早就把设计模式做成了语言特性。
比如 Scala 的单例对象，Python 的 kwargs、函数第一性和内省，哪个不是设计模式的替代？

- 一些设计模式的接口仅仅是为了传递函数，动态语言中的函数是对象，因此不需要。

- 动态语言的对象可以增减自身属性，所以一些类似包装器的模式也是没必要的。

- 一些动态语言已经内置了某个模式，比如 python 的关键字参数和构造器模式。
